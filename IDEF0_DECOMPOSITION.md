# IDEF0-декомпозиция: Электронный учебник «Алгоритмы и структуры данных»

---

## Контекстная диаграмма (A-0)

**Функция:** Предоставление интерактивного обучения алгоритмам и структурам данных

**Входы:**
- Данные для входа / регистрации
- Запрос на открытие модуля
- Исходный массив чисел
- Выбранный ответ на вопрос / тест
- Написанный код решения
- Запрос к AI

**Выходы:**
- Подтверждение входа / ошибка входа
- Отображение теории
- Сгенерированный AI-вопрос
- Результат проверки ответа (верно/неверно)
- Анимация работы алгоритма
- Результат теста (баллы, пройден/не пройден)
- Вывод результата выполнения кода
- Отчёт AI-анализа кода
- Обновлённый прогресс

**Управление:**
- Правила валидации данных входа
- Очерёдность модулей
- Критерий прохождения модуля (3 правильных ответа)
- Критерий прохождения теста (70% и выше)
- Ограничения выполнения кода

**Механизмы:**
- Студент
- Модуль аутентификации
- Модуль отображения теории
- Модуль визуализации
- Модуль тестирования
- Редактор кода
- AI-модуль
- База данных

## Декомпозиция A0 (4 модуля)

### Модуль 1: Аутентификация

**Функция:** Предоставить доступ к системе

**Входы:**
- Данные для входа / регистрации

**Выходы:**
- Подтверждение входа (токен доступа)
- Ошибка входа (неверный логин/пароль)
- Ошибка регистрации (имя занято / неверный формат)

**Управление:**
- Правила валидации данных входа

**Механизмы:**
- Студент
- Модуль аутентификации
- База данных

---

### Модуль 2: Теория и AI-опрос

**Функция:** Предоставить теоретический материал и проверить понимание

**Входы:**
- Запрос на открытие модуля
- Выбранный ответ на вопрос AI

**Выходы:**
- Отображение теории
- Сгенерированный AI-вопрос (текст + 4 варианта)
- Результат проверки ответа (верно / неверно + пояснение)
- Статус прохождения модуля (пройден / не пройден)

**Управление:**
- Очерёдность модулей
- Критерий прохождения модуля (3 правильных ответа)

**Механизмы:**
- Студент
- Модуль отображения теории
- AI-модуль
- База данных

---

### Модуль 3: Визуализация и тестирование

**Функция:** Показать работу алгоритма и проверить знания

**Входы:**
- Исходный массив чисел
- Выбранный / введённый ответ на вопросы теста

**Выходы:**
- Анимация работы алгоритма (пошаговая)
- Статистика выполнения (сравнения, обмены)
- Результат теста (баллы, пройден / не пройден)

**Управление:**
- Критерий прохождения теста (70% и выше)

**Механизмы:**
- Студент
- Модуль визуализации
- Модуль тестирования
- База данных

---

### Модуль 4: Практика и AI-анализ

**Функция:** Обеспечить выполнение практических заданий с обратной связью

**Входы:**
- Написанный код решения
- Запрос к AI (подсказка / анализ)

**Выходы:**
- Вывод результата выполнения кода (успех / ошибка)
- Отчёт AI-анализа кода (проблемы, сложность, рекомендации)
- Обновлённый прогресс

**Управление:**
- Ограничения выполнения кода

**Механизмы:**
- Студент
- Редактор кода
- AI-модуль
- База данных

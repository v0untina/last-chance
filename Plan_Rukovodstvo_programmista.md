## 1. Общие сведения о программе

- Полное наименование: платформа электронного учебника «Алгоритмы и структуры данных».
- Назначение программы (1–2 абзаца, можно взять и сократить из `DIPLOMA_DOCUMENTATION.md` → раздел **1. Введение** → 1.1 Описание проекта, 1.3 Основные функции).
- Технологический стек одной таблицей или списком:
  - Frontend: React 18, TypeScript, Vite, Tailwind CSS, Monaco Editor, Zustand, Recharts, dnd-kit, Fuse.js, i18next.
  - Backend: Node.js, Express, TypeScript, Prisma ORM, PostgreSQL.
  - Внешние сервисы: OpenAI API (GPT-4o-mini), GigaChat API, Piston API.
  - Источник: `DIPLOMA_DOCUMENTATION.md` → раздел **2. Технический стек и обоснование** (2.1–2.4).

## 2. Архитектура системы

- Трёхуровневая архитектура: представление (React SPA) → бизнес-логика (Express API) → данные (PostgreSQL).
- Схема взаимодействия компонентов (клиент ↔ сервер ↔ БД ↔ внешние ИИ-провайдеры ↔ Piston API).
- Источник: `DIPLOMA_DOCUMENTATION.md` → раздел **3. Архитектура приложения** (3.1 Структура монорепозитория, 3.2 Схема взаимодействия компонентов, 3.3 Принципы организации кода).
- Можно переиспользовать (с небольшой адаптацией формулировок) описание архитектуры из раздела 2.1 пояснительной записки диплома — там оно уже выверено по реальному коду.

## 7. Интеграция с ИИ

- Архитектура: паттерны Factory + Strategy (`AIProviderFactory`).
- Интерфейс `IAIProvider`, абстрактный класс `BaseAIProvider` (санитизация от prompt injection, общая retry-логика).
- Провайдеры: `OpenAIProvider` (OpenAI API, модель GPT-4o-mini), `GigaChatProvider` (API Сбербанка, OAuth).
- Отказоустойчивость: Circuit Breaker (порог — 3 последовательные ошибки, отключение на 60 секунд), автоматический fallback на резервного провайдера.
- Кэширование: LRU-кэш на 1000 записей (`cache.ts`).
- Системные промпты и сценарии использования (генерация вопроса по теории, анализ кода и т.д.).
- Источник: `DIPLOMA_DOCUMENTATION.md` → раздел 7 (полностью), а также раздел 3.2 пояснительной записки диплома, где это уже описано и сверено с кодом — можно почти напрямую переносить формулировки.
